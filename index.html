KCL:

Introduction to Circuit Theory-1

(1)Nodes, Branches and Loops

  1. To differentiate between a circuit and a network, we may regard a network as an interconnection of elements or devices, whereas a circuit is a network providing one or more closed paths.

  2. A branch represents a single element such as a voltage source or a resistor. In other word, a branch represents any two-terminal element.

  3. A node is the point of connection between two or more branches. If a short circuit (a connecting wire) connects two nodes, the two nodes constitute a single node.(For the sake of clarity)

  4. A loop is any closed path in a circuit. A loop is a closed path formed by starting at a node, passing through a set of nodes, and returning to the starting node without passing through any node more than once.

    A loop is said to be independent if it contains a branch which is not in any other loop. Independent loops or path results in independent sets of equations .

多人协同开发下的文件冲突问题

由上一篇文章写到，多人协同开发时会有多个用户对资源库中的内容进行相关操作。而如果此时“张三”，“李四”，“王五”分别下载同一个文件进行修改，当他们上传修改后的文件时，新提交的文件便会替代后提交的文件，因此产生文件冲突问题。

集中版本控制系统

VSS

方式：通过对资源库文件进行上锁解锁避免文件冲突问题。

当张三下载文件1时，VSS会自动锁定资源库中的文件1，李四和王五无法下载文件1直到张三修改完成并上传文件1。

但是很明显，这种方式在一个时间段只有一个人能对指定文件进行修改，不符合多人协同开发的高效目的，因此VSS最终被淘汰。

CVS/SVN

方式：对多个用户上传的版本进行比对，而不是直接替换。

在这种情况下，三个人都可以下载和修改文件，但是上传之后，他们对不同部分的内容进行操作，然后再进行比对并把修改都应用到资源库中。

但是集中版本控制系统有一个致命问题，就是当中央服务器失效时，用户就无法对数据进行操作，甚至因此损失数据。

分部式版本控制系统

在中央服务器的基础上搭建一个本地资源库，而用户访问的客户端也不再是中央服务器，而是本地资源库，当中央服务器恢复工作时，通过本地资源库和中央服务器的关联将修改应用到中央服务器。

关键词：版本控制；集中式版本控制；分布式版本控制；多人协作开发

  • 版本

    软件版本 文件版本

    JDK 1.8,17,20

  • 文件版本

    1. 保存重要的历史记录
    2. 恢复数据

  • 版本控制软件

    基本功能：

    1. 保存和管理文件
    2. 提供客户端工具进行访问（不直接对版本进行操作）
    3. 提供不同版本文件的比对功能（确认差异和执行修改）

  • 围绕中央服务器进行的多人协作开发

    对于项目数据的管理应该是将所有资源集中到一个资源库进行储存，而不是分布到各个服务器，该服务器称为中央服务器。

    当用户需要对服务器内的数据进行操作时，就需要通过客户端工具从中央服务区把资源下载到本地，当多个用户进行相关操作时，这个工作状态称为多人协同开发。
